# 文件结构说明

> 说明：本文件用于描述项目的核心目录与关键文件职责，涵盖了基础设施（cloud/gateway）以及 AI 模块（模型资产、脚本工具与服务接口），便于开发者快速定位代码。

## 根目录

- `README.md`：项目总体说明。
- `LICENSE`：开源许可证文件。
- `.github/`：GitHub 配置目录。
  - `workflows/`：CI、自动化流程等配置。
  - `pull_request_template.md`：PR 模板。
  - `CODEOWNERS`：代码所有者配置。
- `cloud/`：云端服务（Rust）代码目录。
- `gateway/`：设备端 UDP 网关（Rust）代码目录，负责采集并向云端发送传感器数据。
- `doc/`：项目文档目录。
  - `.gitignore`：文档目录忽略规则。
  - `文件结构说明.md`：当前文件结构说明文档。
  - `贡献流程.md`：贡献流程说明文档。
  - `PR文案模板.md`：PR 描述撰写模板与示例。
  - `ai/`：AI 模块相关文档。
    - `README.md`：AI 模块总体说明。
    - `API说明.md`：AI 推理接口约定。
    - `模型方案.md`：AI 模型路线、任务说明与实验计划。
    - `数据说明.md`：数据集来源与标签处理说明。
    - `部署说明.md`：AI 训练流程与本地推理部署指南。
- `service/`：AI 服务端与推断业务目录。
  - `README.md`：AI 服务端目录功能介绍。
  - `infer.py`：本地推理 CLI 入口脚本。
  - `model_loader.py`：模型加载与运行时预测功能挂载组件。
- `scripts/`：项目根目录各类自动化工作脚本目录。
  - `README.md`：脚本用途与执行方式说明。
  - `*.py`：核心主流程流程脚本（如：数据处理、特征提取与模型训练）。
  - `legacy/`：历史脚本归档（作对比验证与兼容备份，不推荐作为默认流程入口）。
- `models/`：AI 增量模型及配置文件目录。
  - `rice_leaf_classifier/`：水稻叶片分析专门模型目录。
    - `README.md`：模型版本与资产引用说明。
    - `*.pt` / `*.pth` / `*.onnx`（如果被生成）：权重 / 导出文件。
    - `config.json` & `labels.json`：模型运行相关的超参环境及映射标签配置。
- `tests/`：各服务的自动化用例（集成 / 单元）目录，主要由 Pytest 运行。
  - `README.md`：关于测试的范围说明及依赖需求。
- `local_data/` ＆ `outputs/`：不被纳入 Git 历史管理的重度数据目录（含数据集、训练日志等产物）。

## Cloud 模块 (`cloud/`)

- `Cargo.toml` / `Cargo.lock`：Rust 项目配置及全依赖锁定。
- `deploy.sh`：自动化部署辅助脚本。
- `config/`：配置保存挂载点。
  - `sensors.toml`：预分配传感器规则映射与 ACK 类型定义。
- `scripts/`：Cloud 的专项 shell 自动化脚本（比如本地单元和烟雾测试）。
- `src/`：业务后端主逻辑隔离模块。
  - `main.rs`：初始化启动，参数拦截处理与入口注册。
  - `server.rs`：UDP 核心事件收包主循环与转发过滤。
  - `http_server.rs`：后端 HTTP 承载处理模拟及仪表盘 API 端口。
  - `config.rs`：通过 Serde 读取与转置 TOML 为 Runtime 可变结构。
  - `payload.rs`：报文拆分与特征结构提取策略机制。
  - `registry.rs` / `token.rs`：客户端准入记录与通信授权鉴权方案。
  - `model.rs` / `constants.rs` / `cli.rs` / `time_util.rs`：元数据封装与公共支撑。

## 维护建议

- 随着子模块（如：`cloud/`、`gateway/` 或 AI 服务层）的进一步拆分演进，若调整主要模块与服务隔离度，请在此文档同步增加子项清单。
- 新增 AI 侧的技术规格、数据规范等参考类指南，统一置入 `doc/ai/` 或各内部关联文件夹的 `README.md` 避免全局紊乱。
- 保证 `cloud/scripts`、`gateway/scripts`（服务自驱工具）与根目录 `/scripts`（AI/整个工程范畴的数据整理等工具）边界清晰，勿混淆调用环境。
