# 文件结构说明

## 根目录

- `README.md`：项目总体说明。
- `.github/`：GitHub 配置目录。
  - `workflows/`：CI、自动化流程等配置。
  - `pull_request_template.md`：PR 模板。
  - `CODEOWNERS`：代码所有者配置。
- `cloud/`：云端服务（Rust）代码目录。
- `doc/`：项目文档目录。
  - `.gitignore`：文档目录忽略规则。
  - `文件结构说明.md`：当前文件结构说明文档。
  - `贡献流程.md`：贡献流程说明文档。
  - `PR文案模板.md`：PR 描述撰写模板与示例。

## cloud 目录

- `Cargo.toml`：Rust 工程配置与依赖定义。
- `Cargo.lock`：依赖锁定文件。
- `README.md`：cloud 子项目说明。
- `deploy.sh`：部署脚本。
- `config/`：运行配置目录。
  - `sensors.toml`：传感器规则与接收配置。
- `scripts/`：辅助脚本目录。
- `src/`：服务源代码目录（已按职责拆分模块）。
- `target/`：Rust 构建输出目录（本地生成产物）。

## cloud/src 模块说明

- `main.rs`：程序入口，负责参数解析、加载配置、启动服务与测试入口。
- `constants.rs`：默认配置常量与默认值辅助函数。
- `model.rs`：核心数据结构与类型定义（配置模型、运行时模型、规则模型等）。
- `cli.rs`：命令行参数解析与帮助文本输出。
- `config.rs`：TOML 配置解析、校验与运行时配置构建。
- `payload.rs`：UDP payload 解析与规则匹配逻辑。
- `server.rs`：UDP 服务主循环（收包、匹配、ACK 回包、统计）。

## 维护建议

- 新增目录或关键文件后，同步更新本文件。
- 模块职责发生变化时，优先更新 `cloud/src` 模块说明，避免文档与代码不一致。
- 文档命名建议使用清晰、可读的中文或中英双语。
