# 文件结构说明

> 说明：本文件用于描述 `feature/ai_rice_cls_componentize` 分支中的目录与关键文件职责，便于快速定位代码与模型资产。

## 根目录

- `README.md`：项目简介与整体说明。
- `LICENSE`：开源许可证文件。
- `.github/`：GitHub 配置目录。
  - `workflows/`：CI、分支保护与自动化流程。
  - `pull_request_template.md`：PR 模板。
  - `CODEOWNERS`：代码所有者配置。
- `doc/`：项目文档目录。
  - `.gitignore`：文档目录忽略规则。
  - `文件结构说明.md`：当前文件结构说明文档。
  - `贡献流程.md`：贡献流程说明文档。
- `service/`：服务端与业务组件目录。
  - `README.md`：服务模块说明、组件职责与运行方式。
- `scripts/`：脚本与自动化工具目录。
  - `README.md`：脚本用途、执行方式与注意事项。
  - `*.py`：当前主流程脚本（如数据处理、训练、推理、评估等），用于承载组件化后的标准执行入口。
  - `legacy/`：历史脚本目录（兼容保留）。
    - `*.py`：旧版实验/流程脚本，主要用于回溯、对比与兼容性验证，不作为默认主流程入口。
- `models/`：模型资产目录。
  - `rice_leaf_classifier/`：水稻叶片分类模型目录。
    - `README.md`（如有）：模型版本、标签定义与使用说明。
    - `*.pt` / `*.pth` / `*.onnx`（如有）：模型权重或导出文件。
    - `config.*`（如有）：模型推理/训练相关配置。

## 维护建议

- 新增目录或关键文件后，同步更新本文件。
- 目录说明优先写“用途 + 关键内容”，便于快速定位。
- `scripts/` 与 `models/` 下建议按“主流程 / 兼容历史 / 资产说明”三类持续整理。
- 文档命名建议使用清晰、可读的中文或中英双语。
