# 文件结构说明

> 说明：本文档用于描述项目核心目录与关键文件职责，覆盖云端网关、AI 模型资产、推理服务、测试与项目文档，便于开发者快速定位代码。

## 根目录

- `README.md`：项目整体说明。
- `LICENSE`：开源许可证文件。
- `.github/`：GitHub 配置目录。
  - `workflows/`：CI 与自动化流程配置。
  - `pull_request_template.md`：PR 模板。
  - `CODEOWNERS`：代码归属配置。
- `cloud/`：云端服务代码目录，包含 Rust 服务端实现。
- `gateway/`：设备侧 UDP 网关代码目录，负责采集并向云端转发传感器数据。
- `doc/`：项目文档目录。
  - `文件结构说明.md`：当前文件结构说明文档。
  - `贡献流程.md`：贡献流程说明。
  - `PR文案模板.md`：PR 描述模板与示例。
  - `ai/`：AI 模块相关文档。
    - `README.md`：AI 模块总体说明。
    - `API说明.md`：AI 推理接口说明。
    - `模型方案.md`：模型路线、任务说明与实验计划。
    - `数据说明.md`：数据集来源与标签说明。
    - `部署说明.md`：训练流程与部署指引。
- `service/`：AI 推理服务目录。
- `scripts/`：项目级自动化脚本目录。
- `models/`：模型权重与配置目录。
- `tests/`：测试代码目录。
- `local_data/`、`outputs/`：不纳入 Git 管理的大体量数据与产物目录。

## AI 服务模块（`service/`）

- `README.md`：AI 服务目录说明、启动方式与接口示例。
- `main.py`：FastAPI 应用入口，负责应用初始化、CORS、日志、环境变量读取以及模型预加载。
- `infer.py`：本地单图推理 CLI 工具，不依赖 HTTP 服务。
- `api/`：接口层，仅负责暴露 HTTP 路由与请求分发。
  - `__init__.py`：接口包初始化。
  - `v1/`：第一版 API 路由空间。
    - `__init__.py`：`v1` 路由包初始化。
    - `predict.py`：提供 `POST /api/v1/predict` 与 `GET /api/v1/health` 两个端点。
- `adapters/`：输入适配层，负责把文件路径、字节流等外部输入转换为内部统一对象。
  - `__init__.py`：适配器包初始化。
  - `image_adapter.py`：图像加载与解码适配逻辑。
- `core/`：核心推理层，承载纯 Python 推理逻辑，不直接依赖 FastAPI。
  - `__init__.py`：核心推理包初始化。
  - `base_predictor.py`：统一预测器抽象基类。
  - `rice_leaf_classifier.py`：水稻叶片分类模型推理实现。
- `schemas/`：Pydantic 契约层，定义服务内部与接口返回的数据结构。
  - `__init__.py`：Schema 包初始化。
  - `prediction.py`：预测请求与响应相关的数据模型。

## 测试目录（`tests/`）

- `README.md`：测试范围与运行说明。
- `test_api.py`：FastAPI 接口集成测试，覆盖健康检查与预测接口。
- `test_adapter.py`：输入适配层测试。
- `test_schemas.py`：Pydantic 数据结构测试。
- `test_infer_smoke.py`：本地推理命令烟雾测试。
- `test_prepare_dataset.py`：数据准备脚本相关测试。

## 模型与脚本

- `models/rice_leaf_classifier/`：水稻叶片分类模型目录。
  - `README.md`：模型版本与资产引用说明。
  - `config.json`：模型配置。
  - `labels.json`：类别标签映射。
  - `*.pt`、`*.pth`、`*.onnx`：模型权重或导出文件。
- `scripts/README.md`：项目级脚本用途说明。
- `scripts/*.py`：数据处理、特征提取、训练与评估脚本。
- `scripts/legacy/`：历史脚本归档，仅用于兼容与参考。

## Cloud 模块（`cloud/`）

- `Cargo.toml`、`Cargo.lock`：Rust 项目配置与依赖锁定文件。
- `deploy.sh`：部署辅助脚本。
- `config/`：运行配置目录。
  - `sensors.toml`：传感器与 ACK 规则配置。
- `scripts/`：Cloud 模块专用脚本。
- `src/`：云端服务主逻辑。
  - `main.rs`：程序入口与初始化。
  - `server.rs`：UDP 收包、过滤与转发逻辑。
  - `http_server.rs`：HTTP 接口与仪表盘相关逻辑。
  - `config.rs`：配置读取与运行期配置结构。
  - `payload.rs`：报文解析与特征提取。
  - `registry.rs`、`token.rs`：客户端注册与鉴权逻辑。
  - `model.rs`、`constants.rs`、`cli.rs`、`time_util.rs`：公共模型与基础支撑模块。

## 维护建议

- 当 `service/` 内部继续演进时，优先保持 `api / adapters / core / schemas` 的分层边界清晰。
- 新增 AI 接口、模型协议或部署文档时，优先同步更新 `doc/ai/` 下对应说明。
- 如果 `tests/` 新增新的测试分层，请同步补充本文档，避免目录职责失真。
