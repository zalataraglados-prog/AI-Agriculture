# 部署说明

## 架构概览

AI 推理服务以 Docker 容器方式部署，通过 HTTP/JSON 接收 Rust 云端后端的请求。

```
Rust Cloud Backend ──HTTP──▶ AI Engine Container (FastAPI + PyTorch)
                              ├─ POST /api/v1/predict
                              └─ GET  /api/v1/health
```

## 训练环境

- 使用 Kaggle 或本地 Python 环境完成训练
- 安装开发依赖：`pip install -r requirements-dev.txt`

## 推理服务部署

### 方式一：Docker Compose（推荐）

```bash
# 1. 将训练好的模型放到指定目录
cp best_model.pth models/rice_leaf_classifier/best_model.pth

# 2. 构建镜像
docker compose build

# 3. 启动服务
docker compose up -d

# 4. 验证
curl http://localhost:8000/api/v1/health

# 5. 测试推理
curl -X POST http://localhost:8000/api/v1/predict -F "file=@test.jpg"

# 6. 查看日志
docker compose logs -f ai-engine

# 7. 停止
docker compose down
```

### 方式二：本地 uvicorn 直接运行

```bash
# 安装依赖
pip install -r requirements-dev.txt

# 启动
uvicorn service.main:app --reload --host 0.0.0.0 --port 8000
```

## 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `MODEL_CHECKPOINT_PATH` | 模型权重文件路径 | `models/rice_leaf_classifier/best_model.pth` |
| `MODEL_LABELS_FILE` | 标签文件路径 | `models/rice_leaf_classifier/labels.json` |
| `MODEL_CONFIG_FILE` | 模型配置路径 | `models/rice_leaf_classifier/config.yaml` |

## Docker 镜像设计

- **多阶段构建**：builder 阶段安装依赖，runtime 阶段仅复制产物，镜像更精简
- **CPU-only PyTorch**：推理不需要 CUDA，镜像约 ~1.2GB（vs CUDA 版 ~6GB）
- **非 root 用户**：容器内以 `appuser` 运行，安全最佳实践
- **模型 Volume 挂载**：权重文件通过 volume 挂载，换模型无需重建镜像
- **健康检查**：Docker HEALTHCHECK 调用 `/api/v1/health`

## 依赖管理

| 文件 | 用途 |
|------|------|
| `requirements.txt` | 运行时依赖（进 Docker） |
| `requirements-dev.txt` | 开发依赖（含 torch、pytest 等） |

> 注意：`torch` 和 `torchvision` 不在 `requirements.txt` 中，由 Dockerfile 通过 CPU-only 索引单独安装。
